Key Schemas (Pydantic вЂ” Source of Truth)

Event:
  event_id         str (UUID)
  timestamp        datetime
  source           Literal["SEC_EDGAR", "GDELT", "NEWSAPI"]
  url              str
  raw_text         str
  content_hash     str (SHA-256, for dedup)
  entities         list[Entity]
  event_type       str | None
  summary          str | None
  confidence       float | None
  metadata         dict

Entity:
  ticker           str
  cik              str | None
  name             str
  sic_code         str | None

Signal:
  signal_id        str (UUID)
  event_id         str (FK в†’ Event)
  timestamp        datetime
  ticker           str
  signal_type      Literal["volatility_spike", "positive_drift", "negative_drift",
                           "volume_surge", "watch"]
  signal_text      str
  confidence       float (0.0вЂ“1.0, calibrated)
  rationale        str (>50 chars)
  uncertainty      str (>20 chars)
  impact_window    Literal["1h", "4h", "24h", "1w"]
  predicted_move   float | None
  actual_move      float | None
  evidence         list[EvidenceItem]
  citations        list[Citation]
  metadata         dict (model_version, agent_chain, evidence_count, avg_similarity)

EvidenceItem:
  event_id         str
  event_summary    str
  ticker           str | None
  outcome          str
  similarity_score float (0.0вЂ“1.0)
  time_delta       str

Build Order / Dependencies

  1. backend/src/models/        Must exist first вЂ” all other code imports these
  2. backend/src/config.py      Environment config (DB URLs, API keys, etc.)
  3. backend/src/storage/       Database layer вЂ” independently testable
  4. backend/src/ingestion/     Depends on models + storage
  5. backend/src/nlp/           FinBERT, entity resolver вЂ” independently testable
  6. backend/src/rag/           Depends on models + vector store credentials
  7. backend/src/agents/        Depends on ingestion + nlp + rag + storage
  8. backend/src/evaluation/    Depends on storage (reads events/signals)
  9. backend/src/api/           Depends on storage + agents (Phase 2)
  10. backend/src/gateway/      Depends on storage (Phase 2)
  11. frontend/ hooks/types     Depends on API being live (Phase 2)

Notes

  - This skeleton is informed by the AgentPredict project architecture
    review (March 2026). Key patterns adopted:
    1. RAG-powered Impact Hypothesis Agent (vector store for evidence retrieval)
    2. evidence[] on every signal (grounded, trustworthy confidence scores)
    3. Meaningful change gating at ingestion (filter noise before agents)
    4. WebSocket gateway separation (Phase 2, clean real-time architecture)

  - Patterns NOT adopted from AgentPredict (wrong tradeoffs for 2-person team):
    C++, gRPC, ring buffers, service-per-agent Docker topology

  - Frontend V1 already exists on main (landing page, globe, navbar, auth).
    Phase 2 will add signal dashboard components and WS hooks.

  - Every backend module has a corresponding test file.
    No unimplemented logic left silently empty вЂ” TODO markers where needed.
